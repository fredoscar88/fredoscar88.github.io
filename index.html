Changelog:

Hoo boy, is this one helluvan update

  • -Changed syntax handling
  • -Adding ":NEW SET" to a file now starts a new chain of command blocks
  • -"POS" and "PAT" when defined after ":NEW SET" define the position (of the first block) and pattern of that set
  • -If "POS/PAT" aren't specified after ":NEW SET", the "POS/PAT" default to what's in MSP.properties
  • -Due to the logic of the other above features it is now possible to overwrite previous command block lines
  • -Updated line interpreting, fixed a bug regarding the first "block line" of a file
  • -Added ability to change the redstone directory (now you can switch between projects more easily)
  • -Added ability to turn on and off players' ability to run commands, like "!OpMe". This is the OIT setting and defaults to false.
  • -Added ability to add your own MSP enabled commands, inside "cc.txt" in the MSPPlayers directory
  • -Added advanced readme file, including a syntax guide and info on MSP.properties. See this for help withthe above 2 features.
  • -Added "scripting", you can write a list of commands to be executed at once in the Scripts directory
  • -Lines beginning with "//" are now treated as comments
  • -Added better support for identifying players on teams (when they run commands)
  • -Re-wrote, re-worked a lot of legacy code. Currently this is still sitting in the source, but is unused.
  • And probably some other minor details that I've neglected to mention
  • Need to create a comprehensive list of bugs at some point, too.
